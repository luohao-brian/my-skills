# AI 新闻简报

产出中文版 AI 新闻简报。按固定来源扫描、搜索补漏、详情页核对、统一格式成稿。

开始执行前，必须先读：

- [sources.md](sources.md)：来源类型、品牌入口、搜索引擎分工
- [format.md](format.md)：最终输出格式

硬规则：

1. 没有先读 `sources.md`，不要开始抓取。
2. 没有先读 `format.md`，不要开始写结果。
3. 开始写最终 markdown 前，必须再读一遍 [format.md](format.md)。
4. 用户说“今天 / 今日 / 最新 / 刚刚 / 日报”时，只保留目标时间窗内的条目。
5. 如果目标时间窗内有效内容不足，直接说明覆盖不足，不要偷偷放宽时间窗。
6. 每条进入成稿的新闻，都必须落到一个带明确发布时间的详情页、官方发布页或权威报道页。
7. 最终结果必须是中文 markdown 简报，按分类组织，不能输出线性列表。
8. 列表页或详情页抓不到时，用 `tavily` 补；补不到带明确发布时间的详情页时，跳过。

## 默认时间窗口

用户没指定日期时，默认整理最近 `24-48 小时` 内仍有参考价值的 AI 新闻。

- 默认先看最近 48 小时
- 48 小时内内容太少时，只说明覆盖不足，不放宽时间窗
- 用户指定“今日 / 昨日 / 本周”时，按用户要求执行，并在标题里写绝对日期
- 发布时间默认写相对时间；有绝对时间时补在括号里

## 分类

- 大模型
- Agent
- 融资 / 商业
- 安全 / 治理
- 应用 / 产品
- 开源

某个分类在目标时间窗内没有足够有效内容时，可以省略。

## 工作流

1. 用户说“今天 / 今日 / 最新 / 日报”时，先做结构化搜索建候选集。
   - `tavily`、`bocha` 显式带时间范围
   - 国际英文动态补 `brave`
2. 按 [sources.md](sources.md) 扫固定来源。
   - 官网 / 官方发布入口
   - 权威新闻网站
   - 权威论坛 / 社区
   - 新闻聚合网站
   - 开源模型 / 开源项目入口
3. 用搜索补漏和交叉验证。
   - 至少覆盖 `tavily`、`bocha`
   - 英文动态补 `brave`
   - 重点品牌做官方域名搜索
   - 不要只搜 `AI news` 这种宽泛 query
4. 抓详情页并过滤。
   - 只保留 AI 相关内容
   - 同一事件只保留一条
   - 无发布时间、无正文支撑、明显营销软文不收录
   - 不在目标时间窗内的不收录
5. 成稿前再读一遍 [format.md](format.md)。

## 执行清单

执行时按这个顺序自检：

1. 是否已读 `sources.md` 和 `format.md`
2. 是否已扫固定来源并做搜索补漏
3. 是否至少使用了 `tavily`、`bocha`
4. 英文国际动态是否补了 `brave`
5. 每条是否都有带时间的详情页
6. 最终输出前是否重新读取了 [format.md](format.md)
7. “今天 / 最新”场景下，是否每条都在目标时间窗内
