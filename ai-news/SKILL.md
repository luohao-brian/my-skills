# AI 新闻简报

产出中文版 AI 新闻简报。核心不是“搜到一些链接”，而是按稳定 workflow 获取一手来源、用搜索补漏和交叉验证，再按统一格式成稿。

开始执行前，必须先读：

- [sources.md](sources.md)：官方优先源、媒体补充源、搜索引擎分工
- [format.md](format.md)：最终输出格式

硬规则：

1. 没有先读 `sources.md`，不要开始抓取。
2. 没有先读 `format.md`，不要开始写结果。
3. 开始写最终 markdown 前，必须再读一遍 [format.md](format.md)。
4. 如果入口失败、跳转、返回 landing page、返回壳页面或信息价值存疑，先回到 [sources.md](sources.md) 查对应的访问说明、权重和替代入口，不要自己临时决定。

## 基本原则

1. 默认使用最通用的 `web_search` 和 `web_fetch` 做编排，不要假设存在额外专用工具。
2. 先扫官方源，再扫媒体与聚合源，最后用搜索补漏和交叉验证。
3. 成稿时优先引用官网、官方博客、官方产品页、官方社媒，而不是媒体转述。
4. 最终结果必须是中文 markdown 简报，按分类组织，不能输出线性列表。

## 目标

生成一份 markdown 简报，满足：

1. 必须按分类组织，不允许只输出线性列表。
2. 每条新闻必须包含标题、摘要、发布时间、来源链接。
3. 标题和摘要用中文。
4. 同一条新闻只能出现一次，不要跨分类重复。
5. 优先引用官网、官方博客、官方产品页、官方社媒，而不是媒体转述。

## 默认时间窗口

如果用户没有指定日期，默认整理最近 `24-48 小时` 内仍有参考价值的 AI 新闻。

时间判断规则：

- 默认先看最近 48 小时
- 如果 48 小时内信息太少，再明确说明覆盖不足，不要偷偷放宽到更久
- 如果用户指定“今日 / 昨日 / 本周”，按用户要求执行，并在标题里写绝对日期
- 发布时间展示优先用相对时间，例如 `3 小时前`；如果有绝对时间，建议写成 `3 小时前（2026-03-29 14:30）`

## 分类

- 大模型
- Agent
- 融资 / 商业
- 安全 / 治理
- 应用 / 产品
- 开源

如果某个分类在目标时间窗内没有足够有效内容，可以省略；但必须先完成官方源扫描和搜索补漏，不能先入为主地判定“没有新闻”。

## 工作流

### 1. 先扫官方源

先读 [sources.md](sources.md) 里的“官方一级优先源矩阵”和“信息源价值排序”，优先看这些公司的官方页面：

- OpenAI
- Anthropic
- Google AI / Google DeepMind
- xAI
- MiniMax
- 智谱 AI
- Kimi / Moonshot
- 阿里 AI
- 腾讯 AI
- 火山引擎
- Manus

规则：

- 先看官方页面，再看官方社媒，再看媒体报道
- 对 OpenAI、xAI 这类可能有 Cloudflare 或反爬挑战的站点，不要因为简单 HTTP 失败就判定源站失效
- 官方页面抓取优先用 `web_fetch`
- 如果页面内容不全、只拿到壳页面，继续用 `web_search` 对官方域名做定向搜索

### 2. 再扫媒体与聚合源

再读 [sources.md](sources.md) 里的“媒体与聚合补充源矩阵”和“低权重源处理建议”。

这些源只做三件事：

- 补官方源没及时发出的产品更新
- 发现中文侧热点和转载链路
- 补融资、合作、生态消息

只要官方源里已有一手公告，成稿时优先引用官方链接。

### 3. 用搜索做补漏和交叉验证

固定源之后，必须再做搜索补漏。不要停留在“官网 + 媒体”两层。

开始搜索前，先回看 [sources.md](sources.md) 里的“搜索引擎分工”和“最低搜索覆盖要求”，不要临时发明搜索策略。

搜索引擎使用要求：

- `tavily`：做宽召回、时效新闻和深度补漏
- `bocha`：做中文宽度补充和中文站点补漏
- `brave`：做英文国际动态、英文交叉验证和官方账号补充

最低要求：

- 至少覆盖 `tavily`、`bocha`
- 对英文国际动态或需要英文交叉验证的分类，再补 `brave`
- 每个重要分类至少做一轮中文 query 和一轮英文 query
- 对重点公司至少做一轮“官方域名 + 产品名”定向检索

搜索顺序固定为：

1. 官方站点定向检索
2. 分类主题宽召回
3. 日期 / 语言 / 站点定向检索

不要只搜 `AI news` 这种宽泛 query。要拆成：

- 公司名 + 产品名
- 分类主题词
- 日期词面 query

示例：

- `site:openai.com GPT-5.4 OR ChatGPT 2026-03-29`
- `2026-03-29 AI agent news`
- `2026年3月29日 大模型 发布`

### 4. 抓详情页并过滤

候选新闻进入成稿前，要尽量补齐：

- 中文标题
- 中文摘要
- 发布时间
- 最终来源链接

过滤要求：

- 只保留 AI 相关内容
- 同源重复报道只保留信息最完整的一条
- 二手转载优先让位于原始来源或一手报道
- 无发布时间、无正文支撑、明显营销软文不要收录
- 对只有单点来源的信息，至少再做一轮搜索交叉验证

### 5. 成稿前重新读 format.md

开始写最终答案前，必须再次读取 [format.md](format.md)，不要只凭记忆输出。

最低要求：

- 分类是二级标题
- 每条新闻是三级标题
- 每条固定三行字段：`摘要`、`发布时间`、`来源链接`
- 不输出抓取日志、命令回显、过程噪音

## 执行清单

执行时按这个顺序自检：

1. 官方源是否先于媒体源被扫描
2. 是否至少使用了 `tavily`、`bocha`
3. 英文国际动态是否补了 `brave`
4. 详情页字段是否补齐
5. 最终输出前是否重新读取了 [format.md](format.md)
