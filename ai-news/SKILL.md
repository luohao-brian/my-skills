# AI 新闻简报

产出中文版 AI 新闻简报。使用高产新闻入口和新闻聚合入口，按来源数据规则、候选筛选、入稿字段核对和固定格式成稿。

开始执行前，必须先读：

- [sources.md](sources.md)：来源入口和字段预期
- [format.md](format.md)：最终输出格式

硬规则：

1. 没有先读 `sources.md`，不要开始收集来源。
2. 没有先读 `format.md`，不要开始写结果。
3. 用户说“今天 / 今日 / 最新 / 刚刚 / 日报”时，只保留目标时间窗内的条目；内容不足就说明覆盖不足，不放宽时间窗。
4. 入稿条目必须有标题、发布时间、来源 URL 和摘要所需内容。
5. 最终结果必须是中文 markdown 简报，按分类组织，不能输出线性列表。
6. 固定来源不可访问、返回壳页面、无正文或无明确发布时间时，记录来源不可用或候选字段不完整；不要把绕路查找写成 skill 规则。
7. 先按时间窗、AI 相关性和重复事件筛选候选，再核对入稿字段。
8. 开始写最终 markdown 前，必须再读一遍 [format.md](format.md)。

## Agent 边界

执行这个 skill 时：

1. 不要对 [sources.md](sources.md) 中的固定来源做可信度排序、剔除、替换或维护决策。
2. 不要判断固定来源内容的真伪；只检查字段是否满足成稿要求。
3. 可以基于时间窗、AI 相关性、重复事件、分类覆盖和字段完整性筛选候选。
4. 来源不可访问或字段不完整时，记录来源名称、URL 和缺失字段。

## 时间窗口

用户没指定日期时，整理最近 `24-48 小时` 内仍有参考价值的 AI 新闻。

- 先看最近 48 小时
- 48 小时内内容太少时，只说明覆盖不足，不放宽时间窗
- 用户指定“今日 / 昨日 / 本周”时，按用户要求执行，并在标题里写绝对日期
- 发布时间写相对时间；有绝对时间时补在括号里

## 分类

- 模型 / 研究
- Agent / 开发者工具
- 技术博客 / 工程实践
- 产品 / 应用
- 商业 / 融资
- 安全 / 治理

某个分类在目标时间窗内没有有效候选时，可以省略；有有效候选时，不要因综合、主题归并或跨来源去重把分类压得过低。

## 分类容量

日报成稿按下面目标区间控制每类条数。有效候选达到区间下限时，至少输出下限数量；有效候选不足下限时，输出全部有效候选。

- 模型 / 研究：18-28 条
- Agent / 开发者工具：12-20 条
- 技术博客 / 工程实践：6-10 条
- 产品 / 应用：18-30 条
- 商业 / 融资：4-7 条
- 安全 / 治理：6-10 条

去重只合并同一 URL、同一原文转载、同一事件在多个入口重复出现的候选。不要把同一主题下的不同产品、模型、论文、功能、融资或治理事件综合成一条；综合新闻和聚合入口本身已经做过一定筛选，字段完整的独立条目应按条保留。

达到区间上限后，保留时间更新、事件更重要、字段更完整、重复度更低的条目。不要为了填满某个分类放宽时间窗或字段要求。

## 工作流

1. 读取 [sources.md](sources.md) 的日报扫描入口，建立候选集。
   - 只使用 [sources.md](sources.md) 列出的入口
   - 不为了补数量扩展来源范围
   - 任务范围包含官网、官博或技术博客内容时，只按 [sources.md](sources.md) 的定向取数入口取数
2. 筛选候选。
   - 删除过期、非 AI、营销软文、字段不完整，以及同一 URL / 同一原文转载 / 同一事件跨入口重复的条目
   - 同一主题下的不同进展不要合并成一条
3. 核对入稿字段。
   - 标题、发布时间、来源 URL 和摘要所需内容齐全才收录
   - Feed、列表页、日期页或聚合页字段完整时可以直接成稿
4. 成稿前再读一遍 [format.md](format.md)。

## 执行清单

执行时按这个顺序自检：

1. 是否已读 `sources.md` 和 `format.md`
2. 是否只使用 `sources.md` 列出的入口
3. 是否避免为了补数量扩展来源范围
4. 来源不可访问、正文缺失或字段不完整时，是否记录来源不可用或候选字段不完整
5. 是否先筛选候选，再核对入稿字段
6. Feed、列表页、日期页或聚合页字段完整时，是否可直接作为最终 `来源链接`
7. 最终输出前是否重新读取了 [format.md](format.md)
8. “今天 / 最新”场景下，是否每条都在目标时间窗内
9. 是否按分类容量输出有效候选，并避免把同主题不同进展综合成一条
