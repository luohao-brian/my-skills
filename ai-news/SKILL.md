# AI 新闻简报

产出中文版 AI 新闻简报。按固定来源、来源数据规则、候选筛选、入稿字段核对和固定格式成稿。

开始执行前，必须先读：

- [sources.md](sources.md)：来源类型、品牌入口、来源数据预期和不可用报告规则
- [format.md](format.md)：最终输出格式

硬规则：

1. 没有先读 `sources.md`，不要开始收集来源。
2. 没有先读 `format.md`，不要开始写结果。
3. 开始写最终 markdown 前，必须再读一遍 [format.md](format.md)。
4. 用户说“今天 / 今日 / 最新 / 刚刚 / 日报”时，只保留目标时间窗内的条目。
5. 如果目标时间窗内有效内容不足，直接说明覆盖不足，不要偷偷放宽时间窗。
6. 每条进入成稿的新闻，都必须有标题、发布时间、来源 URL 和摘要所需内容。
7. 最终结果必须是中文 markdown 简报，按分类组织，不能输出线性列表。
8. 固定来源不可访问、返回壳页面、无正文或无明确发布时间时，记录来源不可用或候选字段不完整；不要把绕路查找写成 skill 规则。
9. 不要把全部召回候选都发起逐条字段核对任务；先按时间窗、AI 相关性、重复事件和分类覆盖筛掉候选，再只核对准备入稿条目的必要字段。

## Agent 边界

执行这个 skill 时：

1. 不要对 [sources.md](sources.md) 中的固定来源做可信度排序、剔除、替换或维护决策。
2. 不要判断固定来源内容的真伪；只检查字段是否满足成稿要求。
3. 可以基于时间窗、AI 相关性、重复事件、分类覆盖和字段完整性筛选候选。
4. 来源不可访问或字段不完整时，记录来源名称、URL 和缺失字段。

## 默认时间窗口

用户没指定日期时，默认整理最近 `24-48 小时` 内仍有参考价值的 AI 新闻。

- 默认先看最近 48 小时
- 48 小时内内容太少时，只说明覆盖不足，不放宽时间窗
- 用户指定“今日 / 昨日 / 本周”时，按用户要求执行，并在标题里写绝对日期
- 发布时间默认写相对时间；有绝对时间时补在括号里

## 分类

- 大模型
- Agent
- 融资 / 商业
- 安全 / 治理
- 应用 / 产品
- 开源

某个分类在目标时间窗内没有足够有效内容时，可以省略。

## 工作流

1. 用户说“今天 / 今日 / 最新 / 日报”时，先建立候选集。
   - 候选集应覆盖中文来源、英文来源、官方发布入口和技术来源
   - 候选必须具备目标时间范围内的可用时间字段和来源 URL
   - 来源不可用或候选字段不完整时，记录问题，不放宽时间窗
2. 按 [sources.md](sources.md) 扫固定来源。
   - 官网 / 官方发布入口
   - 权威新闻网站
   - 权威论坛 / 社区
   - 新闻聚合网站
   - 开源模型 / 开源项目入口
3. 做来源数据检查。
   - 重点品牌优先使用官方域名、官方账号、官方博客、官方文档或权威报道
   - 社区和聚合入口按 [sources.md](sources.md) 中的数据预期处理
   - 固定来源不可用时，在结果或执行记录中报告来源问题，提示维护者更新 `sources.md`
4. 筛选候选并核对入稿字段。
   - 先合并同一事件，删除明显过期、重复、非 AI、营销软文的候选
   - 每个分类只保留准备入稿的少量候选再核对标题、发布时间、来源 URL 和摘要所需内容
   - 只保留 AI 相关内容
   - 同一事件只保留一条
   - 无发布时间、无来源 URL、无摘要所需内容的不收录
   - 不在目标时间窗内的不收录
5. 成稿前再读一遍 [format.md](format.md)。

## 执行清单

执行时按这个顺序自检：

1. 是否已读 `sources.md` 和 `format.md`
2. 是否已覆盖固定来源类型并记录不可用来源
3. 中文、英文、官方来源和技术来源是否都已覆盖
4. 重点品牌是否使用官方域名、官方账号、官方博客、官方文档或权威报道
5. 正文缺失或页面不可访问时，是否记录来源不可用或候选字段不完整
6. 是否先筛选候选，再只核对准备入稿条目的必要字段
7. 最终输出前是否重新读取了 [format.md](format.md)
8. “今天 / 最新”场景下，是否每条都在目标时间窗内
