# AI 新闻来源合同

`references/sources.json` 是执行入口；本文只解释字段语义和来源边界。

## 来源边界

- 只消费 `sources.json` 中注册的来源，不在运行中扩展、替换或排序来源。
- `adapter` 决定取数方式；RSS、JSON API 和 HTML 索引页都必须归一化成候选字段。
- HTML 索引页不是 fallback。标记为 HTML 的来源应直接解析页面索引、日期页或嵌入数据。
- 日期页型来源的目标日期页不存在时，可以使用目标日期前后 `72` 小时内最近的已发布日期页建立候选；入稿仍按条目的发布时间、页面日期和目标时间窗筛选。

## 候选字段

每条候选进入筛选前应尽量提供：

- `title`：Feed、列表卡片、日期页条目或文章标题。
- `source_url`：Feed 链接、列表卡片链接、日期页原文链接；没有单条原文链接时可使用聚合日期页链接。
- `published_at`：Feed 时间、列表展示时间、日期页日期或文章发布时间。
- `summary_basis`：Feed 描述、列表卡片描述、日期页摘要、正文片段或文章开头摘要。
- `source_id`：来自 `sources.json` 的稳定来源 id。

## 来源失败

来源不可用、解析失败或字段缺失时，脚本必须记录到 `source_coverage` 或 `missing`。不要只把失败写进日志。
